Documentacion del Proyecto MTTP Arandano

Version del documento: 1.0 | Fecha: 14/04/2026

Resumen ejecutivo

MTTP Arandano es una aplicacion web para registrar, validar, previsualizar y enviar datos operativos de Campo, Packing, Thermo King y Recepcion C5. El sistema soporta trabajo con y sin internet, cola local de pendientes, y generacion de PDF por formato. Para Agrovision Peru S.A.C., esta solucion puede presentarse tanto como herramienta operativa actual como base formal de evolucion tecnica para el area de Desarrollo de Software.

1. Objetivo del sistema

  • Digitalizar el registro de mediciones por ensayo y fecha.
  • Reducir errores de captura mediante validaciones de negocio.
  • Garantizar trazabilidad de datos enviados y pendientes.
  • Permitir operacion offline con sincronizacion posterior.

2. Alcance funcional

Modulo Funcionalidad principal Salida
Campo Captura por ensayo de pesos, tiempos, temperatura, humedad, presion y observaciones. Registro en hoja + PDF de Calibrado Visual.
Formato Packing Captura de tiempos y pesos de proceso con validaciones por orden y limites. Registro en hoja + PDF de Packing.
Thermo King Captura de tiempos/pesos/temperatura/humedad/presion con reglas de Camara MP. Registro en hoja + PDF Thermo King.
Recepcion C5 Registro especializado para C5 con bloqueo por estado y consistencia de filas. Actualizacion en hoja + PDF Recepcion C5.

3. Arquitectura tecnica

  • Frontend: index.html, style.css, script.js.
  • Servicios de red/cache: network.js.
  • Backend: Google Apps Script en Code.gs.
  • Persistencia local: localStorage (cola Campo y cola Packing/C5).
  • Canal servidor: endpoint Apps Script (JSONP para GET, POST para guardado).

4. Tecnologias utilizadas (estado actual)

  • Frontend web: HTML5, CSS3, JavaScript.
  • Backend: Google Apps Script (Code.gs) sobre Google Sheets.
  • Comunicacion: JSONP para consultas GET y POST HTTP para guardado.
  • Persistencia local: localStorage para cache y colas offline.
  • UX/Alertas: SweetAlert2 para mensajes de validacion y estado.
  • PDF: jsPDF (incluyendo operacion offline con libreria local).

5. Operacion sin internet

  • Si no hay red, los datos se guardan en cola local (pendiente).
  • Al recuperar conexion, el sistema sincroniza automaticamente.
  • El estado visual de red y pendientes se actualiza en interfaz.
  • La app mantiene cache de consultas para continuidad operativa.

6. Flujo de uso recomendado

  1. Seleccionar fecha y ensayo.
  2. Cargar datos base de hoja cuando aplique.
  3. Completar registros por wrapper respetando validaciones.
  4. Previsualizar PDF.
  5. Guardar/Enviar (o encolar si no hay red).
  6. Verificar confirmacion y pendientes.

7. Seguridad y calidad de datos

  • Prevencion de duplicados por fecha y ensayo.
  • Validaciones previas en UI y validacion de consistencia por filas.
  • Trazabilidad por ensayo y modulo.
  • Manejo de errores con feedback claro (SweetAlert).

8. Evaluacion de migracion a OneDrive / SharePoint

La empresa puede evaluar migrar el repositorio documental y la capa de almacenamiento operativo a ecosistema Microsoft (OneDrive / SharePoint). Esta decision debe considerar impacto tecnico, seguridad, costo y continuidad de operacion.

Escenario Ventajas Riesgos / Ajustes requeridos
Mantener modelo actual (Apps Script + Sheets) Operativo hoy, bajo esfuerzo adicional, alta velocidad de implementacion. Dependencia Google Workspace para backend y procesos de integracion.
Migrar a SharePoint/OneDrive + stack Microsoft Alineacion con plataforma corporativa, gobierno de datos centralizado en Microsoft 365. Requiere rediseño de backend/API, autenticacion, permisos y procesos de sincronizacion.
Modelo hibrido (corto plazo) Permite operar sin frenar negocio mientras se disena migracion formal. Mayor complejidad temporal por coexistencia de dos entornos.

Recomendacion ejecutiva

Se recomienda presentar MTTP Arandano como prototipo formal funcional para Agrovision Peru S.A.C. y para el area de Desarrollo de Aplicaciones/Software, con decision en dos vias:

  • Via A: Continuidad operativa inmediata sobre stack actual.
  • Via B: Plan de migracion por fases a SharePoint/OneDrive segun lineamientos TI corporativos.

Si por recomendaciones adicionales, lineamientos de seguridad mas amplios o ajustes corporativos se solicita mayor fortalecimiento, el proyecto puede mantenerse como prototipo formal y ser mejorado por el area de Desarrollo de Software sin perder el avance ya validado funcionalmente.

9. Diferencia de despliegue: App movil vs App web instalable

Modalidad Caracteristica Implicancia para la empresa
Aplicacion movil tradicional (Play Store) Se descarga e instala desde tienda, requiere ciclo de publicacion y versionado de app movil. Mayor esfuerzo de distribucion y mantenimiento por plataforma movil.
Aplicacion web instalable (navegador) Corre en web y puede instalarse desde el navegador como acceso directo tipo app. Despliegue mas rapido, actualizacion centralizada y menor friccion operativa.

Para Agrovision Peru S.A.C., el enfoque web instalable permite acelerar adopcion operativa y mantener flexibilidad para una evolucion posterior hacia un producto empresarial mas amplio.

10. Recomendaciones de mantenimiento

  • Mantener versionado de reglas de negocio en cambios de proceso.
  • Realizar pruebas E2E online/offline por cada ajuste importante.
  • Documentar en changelog los cambios de validacion y PDF.
  • Definir checklist de salida para despliegues productivos.
